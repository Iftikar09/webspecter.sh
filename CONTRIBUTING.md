# Contributing to WebSpecter

Thanks for your interest in contributing.

## Philosophy
WebSpecter prioritizes:
- Reasoning over automation
- Evidence over assumptions
- Safety over aggressiveness

## What We Accept
- Logic & reasoning improvements
- Better analyst guidance
- UX clarity & documentation
- Stability & correctness fixes

## What We Do NOT Accept
- Auto exploitation modules
- Blind fuzzers
- Illegal automation
- Payload dumping tools

## Code Standards
- Bash: defensive, strict mode
- Python: standard library only
- Clear comments > clever code

## How to Contribute
1. Fork the repository
2. Create a feature branch
3. Make changes
4. Open a pull request with explanation

All contributions are reviewed with an **ethical security mindset**.
